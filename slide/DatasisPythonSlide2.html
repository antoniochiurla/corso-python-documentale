Anatomia di un Notebook

Un Notebook è composto da Celle. Ne esistono principalmente di due tipi:

  1. Celle di Codice: Dove scriviamo Python. L'output (testi, grafici, immagini) appare subito sotto.
  2. Celle Markdown: Dove scriviamo la documentazione. Supportano titoli, grassetti, liste e persino formule matematiche.

Il concetto di "Kernel":
È il "motore" Python che gira in sottofondo. Ricorda tutte le variabili definite nelle celle precedenti, anche se le celle vengono eseguite in ordine sparso.

Comandi Rapidi e Workflow

Per essere produttivi in Jupyter bastano pochi comandi da tastiera:

  • Shift + Invio: Esegue la cella corrente e passa alla successiva.
  • Alt + Invio: Esegue la cella e ne crea una nuova sotto.
  • Esc + A / B: (In modalità comando) Inserisce una cella sopra (Above) o sotto (Below).
  • Esc + M: Trasforma una cella di codice in Markdown.
  • Esc + DD: Elimina la cella corrente.

Suggerimento: Se usi VS Code, puoi creare un file con estensione .ipynb per avere la stessa esperienza senza uscire dal tuo editor preferito.

Gestione moderna del File System

Dimentichiamo il vecchio modulo os.path. Pathlib tratta i percorsi come oggetti, non come stringhe.
Perché usarlo?

  • Sintassi intuitiva (usa l'operatore / per unire i percorsi).
  • Compatibilità cross-platform automatica (Windows vs Linux).
  • Metodi potenti per estensioni e nomi file.

File System con Pathlib

Esempi: 

from pathlib import Path

base_path = Path("in")
input_file = base_path / "Ricevute.pdf"

print(input_file.name)      # Ricevute.pdf
print(input_file.stem)      # Ricevute
print(input_file.suffix)    # .pdf

Navigazione e Scansione Cartelle

Trovare tutti i documenti in una cartella (e sottocartelle) è immediato con glob e rglob. 

cartella_input = Path("./in")

# Cerca solo i PDF nella cartella corrente
percorsi_pdf = list(cartella_input.glob("*.pdf"))

# Cerca PDF, JPG e PNG in modo ricorsivo
estensioni = ("*.pdf", "*.jpg", "*.png")
tutti_i_files = []
for ext in estensioni:
    tutti_i_files.extend(cartella_input.rglob(ext))

print(f"Trovati {len(tutti_i_files)} documenti.")

PyMuPDF (fitz)

PyMuPDF è una delle librerie più veloci e complete per manipolare i PDF.
Operazioni principali:

  1. Leggere il testo (se presente come "testo nativo").
  2. Estrarre metadati (autore, data creazione).
  3. Renderizzare le pagine in immagini (fondamentale per l'OCR).

pip install PyMuPDF

Il coltellino svizzero per i PDF

# Estrazione testo rapida (se il PDF non è una scansione)

Rendering PDF per l'OCR (Alta Qualità)

Se il PDF è una scansione, possiamo interpretare con OCR.
PyMuPDF ricorre a Tesseract

L'OCR richiede risoluzione!

Dobbiamo impostare una Matrix per aumentare i DPI (standard 300 DPI). 

doc = fitz.open("in/Ricevute.pdf")
pagina = doc[0] # Prendi la prima pagina

# Aumenta la risoluzione (2.0 = 144 DPI, 4.0 = 288 DPI)
zoom = 4
mat = fitz.Matrix(72 * zoom / 72, 72 * zoom / 72)
clip=(15,20,160,50)
pix = pagina.get_pixmap(matrix=mat, clip=clip)

# Salvataggio per il prossimo step (Preprocessing)
pix.save(f"out/Ricevute_zoom{zoom}.png")

Provare con esempio Jupyter

Considerazioni sui DPI

Per definizione 1 pollice contiene 72 punti.
Aumentare lo zoom aumenta la risoluzione.

Se il PDF è "nativo" contiene font vettoriali e "zoommabile" all'infinito.

Zoommando un documento scannerizzato i dettagli non si possono "inventare".
Interpolando si ammorbidiscono i bordi dei pixel.
I software OCR richiedono una risoluzione minima dei caratteri, esempio altezza da 20 a 40 pixel.
Seguono approfondimenti sull'interpolazione nella lezione 3

Pillow metodo convert

image.convert(mode=None, matrix=None, dither=None, palette=0, ...)

mode:
L: 8-bit scala di grigi.
RGB: 24-bit true color.
RGBA: 32-bit true color con trasparenza.
CMYK: 32-bit color separation, utile per la stampa.
YCbCr: 24-bit color video format, solitamente usato per video encoding.
I: 32-bit signed integer, fornisce alta precisione.
F: 32-bit floating point, offre precisione e un'ampia gamma dinamica.

Caso d'Uso: Gestione Documenti Storici

Per scannerizzazioni di documenti stampati o scritti a mano, la risoluzione è tutto.

  • Problema: Immagini troppo pesanti bloccano le API.
  • Soluzione: Ridimensionare mantenendo le proporzioni.
def ridimensiona_proporzionale(image_path, max_width=2000):
    with Image.open(image_path) as img:
        w_percent = (max_width / float(img.size[0]))
        h_size = int((float(img.size[1]) * float(w_percent)))
        img = img.resize((max_width, h_size))
        img.save("ottimizzato.png", "PNG", quality=95)

ridimensiona_proporzionale("scansione_pesante.png")
