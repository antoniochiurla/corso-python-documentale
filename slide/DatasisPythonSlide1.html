Tipi di Dati Base: Caratteristiche e Limiti

  • int: Precisione arbitraria (gestisce numeri enormi senza overflow), utile per contatori di pagine o ID.
  • float: Segue lo standard IEEE 754. Attenzione ai limiti di precisione nei calcoli finanziari (es. 0.1 + 0.2 != 0.3). Per le fatture useremo spesso il tipo Decimal.
  • str: Immutabili. Fondamentali per l'output OCR. Ogni manipolazione crea una nuova stringa in memoria.
  • bool: Fondamentali per i flag di stato (es. is_processed).

Collezioni: List e Tuple

  • List (Liste): Ordinate e mutabili. Ideali per una sequenza di pagine o una lista di file da processare.
pagine = ["pag1.jpg", "pag2.jpg"]
pagine.append("pag3.jpg")
  • Tuple: Ordinate ma immutabili. Più veloci delle liste. Si usano per dati che non devono cambiare (es. coordinate di un box OCR o parametri di configurazione).
box_ocr = (10, 10, 100, 50) # x, y, w, h

Collezioni: Set e Dizionari

  • Set: Collezione non ordinata di elementi unici. Utili per rimuovere duplicati velocemente o per test di appartenenza (molto performanti).
estensioni_ammesse = {"pdf", "png", "jpg"}
  • Dict (Dizionari): Coppie chiave-valore. È la struttura principe per i dati estratti dai documenti.
dati = {"partita_iva": "12345678901", "totale": 150.50}

Controllo di Flusso: Branching

  • if / elif / else: La struttura classica.
  • match (Python 3.10+): Potente per gestire diversi tipi di documenti o stati.
match documento.tipo:
    case "Fattura":
        processa_fattura()
    case "DDT" | "Bolla":
        processa_trasporto()
    case _:
        logger.error("Tipo sconosciuto")

Controllo di Flusso: for

  • for: Per iterare su collezioni, file, pagine... Usare enumerate() se serve l'indice.
frutti = ["mela", "banana", "ciliegia"]

for frutto in frutti:
	print(frutto)

Controllo di Flusso: while

  • while / else: Il blocco else viene eseguito solo se il ciclo termina naturalmente (senza break). Utile per i tentativi di connessione API.
tentativi = 0
while tentativi < 3:
    if chiama_api(): break
    tentativi += 1
else:
    # Eseguito solo se i 3 tentativi falliscono
    alert_admin()

List comprehension

Offre una sintassi sintetica per generare una lista basata sui valori di un'altra lista. 

fruits = ["apple", "banana", "cherry", "kiwi", "mango"]
fruits_with_a = [x for x in fruits if "a" in x]
print(fruits_with_a)

OOP

  • Programmazione orientata agli oggetti
  • Uno stile di programmazione che modella il software attorno a "oggetti" del mondo reale
  • Classe - definisce le proprietà e i metodi di un gruppo di oggetti
  • Oggetto - l'istanza concreta della classe

Incapsulamento

  • Nascondere dei dettagli interni
  • Aiuta a proteggere da usi non ortodossi di proprietà e metodi

Ereditarietà e Polimorfismo

  • Ereditarietà: Una classe "figlia" prende tutto dalla classe "padre" e aggiunge qualcosa di specifico
    • Esempio: La classe ClasseDerivata eredita da NuovaClasse ma aggiunge la proprietà prop3
  • Polimorfismo: La capacità di oggetti diversi di rispondere allo stesso comando in modi diversi
    • Esempio: Se chiamo il metodo .notifica(), per un messaggio normale si invia una mail, per uno urgente si invia un SMS. Il comando è lo stesso, il risultato cambia.

Astrazione

  • Mostrare il lato semplice degli oggetti nascondendo la complessità
  • Si realizza con classi astratte e interfacce

Overriding e overloading.

  • Overriding: Quando la classe figlia riscrive un metodo del padre perché deve fare qualcosa di diverso
  • Overloading: Avere più metodi con lo stesso nome ma che accettano parametri diversi (es. calcola(x) e calcola(x, y)). Nota: In Python puro non esiste come in Java/C#, si gestisce con i parametri opzionali.

Generare Eccezioni (raise)

Non aspettare che Python segnali un errore (es. FileNotFound). Segnala tu quando i dati non sono validi.

def verifica_partita_iva_italiana(partita_iva:str): 

   if not partita_iva.startswith("IT"):
      raise ValueError("Documento estero non supportato")

Lanciare un'eccezione interrompe il flusso e lo delega al primo blocco except disponibile.

Catturare Eccezioni (try/catch)

Racchiudi le istruzioni che possono fallire in un blocco try/catch 

  try:
      verifica_partita_iva_italiana(partita_iva)
  catch ValueError as value_error:
      print("La partita iva non e' italiana")
  else:
      print("La partita iva e' italiana")
  finally:
      print("Ho controllato")

Eccezioni Personalizzate

Usare eccezioni generiche come ValueError rende difficile capire chi ha generato l'errore in uno script complesso.
Definire classi proprie permette di:

  1. Filtrare gli errori in modo mirato.
  2. Trasportare metadati aggiuntivi (es. l'ID del documento).
  3. Rendere il codice auto-esplicativo.

Come definire Eccezioni Personalizzate

Si eredita dalla classe base Exception. 

class DocumentQualityError(Exception):
    """Lanciata quando l'immagine è
    troppo sfocata per l'OCR."""
    def __init__(self, message, score):
        super().__init__(message)
        self.score = score

# Utilizzo
if quality_score < 0.5:
    raise DocumentQualityError("Qualità insufficiente",
                quality_score)

Architettura Modulare

Un modulo è un singolo file con estensione .py
Dividere il codice per responsabilità:

  • main.py: Orchestratore.
  • exceptions.py: Tutte le tue classi di errore custom.
  • core/: Cartella per i moduli di pulizia e OCR.
  • connectors/: Per SharePoint e API esterne.

Moduli

Per definire un nuovo modulo si crea un file con estensione py.
Nel nuovo modulo si aggiungono variabili, funzioni e classi.
Alla fine if __name__ == "__main__": main()
Per accedere ai contenuti di altri moduli va usata l'istruzione import.
Per installare moduli di librerie disponibili nell'ampio catalogo disponibile si usa il comando pip.

Virtual environment

Rappresenta un ambiente Python virtuale completo.
Si usa per differenziare versione di Python e il set di librerie disponibili da quelle dell'installazione standard del linguaggio.
Si crea con il comando python -m venv <directory_virtual_env>
Si attiva con:

  • windows
    • command prompt
      • <directory_virtual_env>\Scripts\activate.bat
    • power shell
      • <directory_virtual_env>\Scripts\Activate.ps1
    • linux
      • source <directory_virtual_env>/bin/activate

Installazione librerie

Molti moduli sono resi disponibili dall'installazione normale di Python.
Per installare altre librerie e renderle disponibili all'interprete Python si usa il comando pip.
pip install <nome_libreria>

Per trovare librerie nel catalogo: https://pypi.org/search

Per definire l'elenco delle dipendenze di un progetto con le rispettive versioni si può realizzare un file requirements.txt da utilizzare con il comando:
pip install -r requirements.txt

Package

Un package è una directory che contiene moduli, deve contenere un file __init__.py e viene eseguito al primo import

Git - concetti di base

Dalla pagina di manuale: "git - the stupid content tracker".
Lo strumento di versionamento più diffuso.
In locale viene creata una directory .git nella directory del progetto con le informazioni interne.
La directory del progetto si chiama "working tree".
Il repository è il contenitore della configurazione e tutta la storia del progetto.
La staging area o index è quel contenitore virtuale dove si mettono le modifiche da includere nel prossimo commit.
Un commit rappresenta un gruppo di modifiche su uno o più file.
I branch rappresentano la storia delle versioni del progetto.
I branch possono avere un branch di riferimento come upstream.

Git - distribuito

Un progetto può essere semplicemente locale oppure può essere collegato ad uno o più repository remoti, di solito uno solo.
Spesso i branch sono collegati ai propri corrispondenti upstream di repository remoti.

Git - aggiornamenti

Quando un branch ha un branch di upstream si eseguono particolari operazioni per sincronizzare le modifiche in ingresso e in uscita.
Per ricevere modifiche da un altro branch si può eseguire una operazione di "merge" per accogliere le nuove modifiche.
In alternativa, se nel nostro branch abbiamo aggiunto modifiche dopo l'ultimo merge, quindi si trova disallineato, possiamo ricorrere a "rebase" che parcheggia le nostre modifiche, applica le nuove modifiche e riapplica le nostre.

Git - stato pulito e allineato

Quando si chiede lo stato del progetto si ottengono le informazioni su:

  • working tree
    • se ci sono modifiche
  • staging area
    • se ci sono file modificati segnati per il commit
  • stato del branch rispetto a upstream
    • se il branch corrente è avanti, indietro o disallineato

Git - operazioni preliminari

  • inizializzazione progetto
    • git init # crea il repository locale
    • git clone <url_repo_remoto> # scarica un repository remoto e crea il working tree
  • configurazioni utente
    • git config user.name "<Il Tuo Nome>" # imposta i dati utente per i commit futuri
    • git config user.email "<indirizzo_e-mail>"

Git - comandi locali

  • I comandi locali frequenti
    • git status # per ottenere lo stato del repository locale
    • git log # per la storia dei commit
    • git checkout <nome_branch> # fa switch verso il branch
    • git checkout -b <nuovo_branch> # crea il branch
    • git branch -u <nome_branch> # collega branch <nome_branch> come upstream
    • git add <file>... # aggiunge i file nell'area di staging
    • git commit -m "commento" # aggiunge il commit
    • git merge <branch_origine> # integra <branch_origine> sul corrente
    • git rebase <branch_base> # riapplica i commit sulla base di <branch_base>

Git - annullare modifiche

  • git restore <file> # annulla le modifiche a <file>
  • git revert <id_commit>... # applica le modifiche inverse del commit o dei commit
  • git reset <id_commit | branch | tag>
    • --soft # file non modificati e inseriti in staging area
    • --mixed # file non modificati ma NON inseriti in staging area
    • --hard # annulla tutto: file e staging area

Git - interazioni con branch remoti

  • Interazioni con i repo remoti
    • git remote add <nome_origin> <url_repo> # <nome_origin> di solito: origin
    • git branch -u <nome_origin>/<nome_branch> # collega branch a origin come upstream
    • git fetch # scarica novità da repo remoto
    • git merge # integra il branch di upstream
    • git pull # combinazione di fetch e merge del branch in origin
    • git push # spedisce novità verso repo remoto

Git - i conflitti

Nelle operazioni di merge, rebase, revert... possono verificarsi dei conflitti
L'operazione si interrompe e servono interventi manuali per risolvere i conflitti
Si correggono i file con i conflitti e si aggiungono alla staging area
Operazioni possibili
--continue # si prosegue dopo aver risolto i conflitti
--skip # salta l'applicazione del commit corrente
--abort # annulla l'intera operazione e torna allo stato precedente
--quit # interrompe l'operazione ma non ripristina lo stato precedente
Si possno presentare numerosi conflitti durante un'operazione di merge o rebase

Git - regole d'oro

  • Di solito si evita di modificare la storia di un branch, specialmente se remoto e condiviso con i colleghi
  • Meglio non mettere le mani dentro .git
  • Da principianti quando si presenta un problema irrisolvibile si cancella il progetto e si fa di nuovo git clone
  • Con l'esperienza il problema si risolve
  • Si usa l'opzione --force solo in casi estremi
